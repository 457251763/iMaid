# iMaid v0.3 Live2D 渲染版本规格说明书

## 1. 概述

### 1.1 背景与目标

v0.2 版本已成功实现悬浮窗口的基础功能，包括窗口置顶、透明背景、拖拽移动和系统托盘集成。v0.3 版本将聚焦于 **Live2D 模型渲染**，为桌宠赋予动态视觉形象，使用户能够看到角色的动画效果和交互反应。

### 1.2 核心目标

- 实现 Live2D 模型的加载与渲染
- 建立完整的动画系统，支持idle、说话、眨眼等基础动画
- 实现用户与角色的交互响应（点击、视线跟随）
- 与现有悬浮窗口架构无缝集成

## 2. 技术方案

### 2.1 技术选型

| 技术 | 选择 | 说明 |
|------|------|------|
| Live2D 渲染库 | pixi-live2d-display | 基于 PIXI.js，成熟稳定 |
| 渲染引擎 | PIXI.js v7+ | WebGL 加速渲染 |
| 模型格式 | Cubism 4.x (.model3.json) | 跨版本兼容性好 |
| 动画控制 | Cubism Native SDK | 原生动画控制 |

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 模型加载

| 需求 | 描述 |
|------|------|
| 模型配置 | 支持从 `public/models/` 目录加载 Live2D 模型 |
| 配置文件 | 读取 `.model3.json` 格式的模型描述文件 |
| 错误处理 | 模型加载失败时显示占位图标并记录日志 |
| 预加载 | 启动时预加载模型，提升首帧渲染速度 |

#### 3.1.2 渲染控制

| 需求 | 描述 |
|------|------|
| 渲染容器 | 在 `HomePage.jsx` 的 Live2D 容器中渲染 |
| 画布尺寸 | 自适应容器尺寸，默认 280x280 像素 |
| 渲染模式 | WebGL 优先，回退到 Canvas 2D |
| 背景透明 | 模型背景保持透明，与窗口透明效果兼容 |

#### 3.1.3 动画系统

| 动画类型 | 触发条件 | 持续时间 |
|----------|----------|----------|
| idle | 无操作时循环播放 | 永久循环 |
| speaking | AI 回复时播放 | 对话期间 |
| tap | 用户点击模型 | 单次播放 |
| blink | 随机触发 | 单次播放 |

#### 3.1.4 交互响应

| 交互类型 | 响应行为 |
|----------|----------|
| 点击模型 | 播放 tap 动画 + 随机台词 |
| 鼠标悬停 | 视线跟随鼠标位置 |
| 长按 | 播放特殊动画 + 打开快捷菜单 |
| 双击 | 打开设置页面 |

## 6. 验收标准

### 6.1 功能验收

| 测试项 | 验收条件 |
|--------|----------|
| 模型加载 | 模型文件存在时正确加载显示 |
| 渲染显示 | 模型在容器中央正确显示，无变形 |

## 8. 风险与限制

### 8.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| WebGL 不可用 | 模型无法渲染 | 提供 Canvas 2D 回退方案 |
| 模型文件损坏 | 启动失败 | 显示占位图标，提示错误 |
| 内存占用过高 | 应用卡顿 | 优化渲染参数，控制模型面数 |

### 8.2 已知限制

- 模型必须使用 Cubism 4.x 格式
- 模型纹理大小建议控制在 2048x2048 以内
- Windows 7 等老系统 WebGL 支持可能受限
